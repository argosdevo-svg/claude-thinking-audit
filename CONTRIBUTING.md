# Contributing

Thanks for your interest in improving this project.

## Quick start

1. Fork the repo and create a feature branch.
2. Make your changes.
3. Verify your changes locally.
4. Open a pull request.

## Development setup

- Python 3.11+ is recommended.
- Install dependencies:

  `python3 -m pip install -r requirements.txt`

## Project structure (high level)

- `addon/` - mitmproxy addons and web UI
- `docs/` - documentation and reports
- `templates/` - UI templates and assets

## Tests

If you change behavior, please include a brief manual validation or test notes
in your PR description.

## Pull request checklist

- The change is scoped and explained.
- The behavior is validated (notes included).
- The docs are updated if user-facing behavior changes.

## Reporting security issues

Please do not open public issues for security reports. See `SECURITY.md`.
